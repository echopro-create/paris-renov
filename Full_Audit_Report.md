## 1. БЕЗОПАСНОСТЬ

### ✅ Исправлено: XSS-уязвимость в API контактной формы
- **Файл:** `api/contact.ts`
- **Проблема:** Данные из формы (имя, email, телефон, сообщение) вставлялись в HTML-письмо без экранирования. Злоумышленник мог внедрить вредоносный HTML/JavaScript в письмо, которое получает владелец сайта.
- **Исправление:** Добавлена функция `escapeHtml()`, которая экранирует все пользовательские данные перед вставкой в HTML-шаблон письма.
- **Серьёзность:** 🔴 Критическая

### ✅ Исправлено: Некорректный canonical URL в security.txt
- **Файл:** `public/.well-known/security.txt`
- **Проблема:** Canonical URL указывал на `atelier-alexei.fr` вместо `da-bat.com`
- **Исправление:** Заменён на `https://da-bat.com/.well-known/security.txt`
- **Серьёзность:** 🟡 Высокая

### ⚠️ Рекомендация: CSP содержит 'unsafe-eval'
- **Файл:** `vercel.json` (строка 28)
- **Проблема:** Директива `script-src` включает `'unsafe-eval'`, что ослабляет защиту от XSS.
- **Рекомендация:** Убрать `'unsafe-eval'` после того, как все зависимости перестанут его требовать. Некоторые библиотеки (например, старые версии) могут зависеть от `eval()`.
- **Серьёзность:** 🟡 Средняя

## 2. КОНФИГУРАЦИЯ И ОКРУЖЕНИЕ

### ✅ Исправлено: .env.example — устаревшие переменные EmailJS
- **Файлы:** `.env.example`, `.env.local.example`
- **Проблема:** В файлах были переменные для EmailJS (`VITE_EMAILJS_SERVICE_ID`, `VITE_EMAILJS_TEMPLATE_ID`, `VITE_EMAILJS_PUBLIC_KEY`), но проект использует Resend. Отсутствовала переменная `RESEND_API_KEY`.
- **Исправление:** Заменены на актуальные переменные: `RESEND_API_KEY`, `VITE_TURNSTILE_SITE_KEY`, `TURNSTILE_SECRET_KEY`.
- **Серьёзность:** 🔴 Критическая

### ✅ Исправлено: Supabase — некорректное приведение типов
- **Файл:** `lib/supabase.ts`
- **Проблема:** Использовался `(import.meta as any).env` — обход системы типов TypeScript.
- **Исправление:** Заменено на `import.meta.env.VITE_SUPABASE_URL` — корректное обращение к типизированным переменным окружения.
- **Серьёзность:** 🟡 Высокая

### ✅ Исправлено: TypeScript strict mode
- **Файл:** `tsconfig.json`
- **Проблема:** Отсутствовали строгие проверки TypeScript: `strict`, `noUnusedLocals`, `noUnusedParameters`, `noFallthroughCasesInSwitch`, `forceConsistentCasingInFileNames`.
- **Исправление:** Добавлены все строгие опции для повышения качества кода и предотвращения ошибок.
- **Серьёзность:** 🟡 Средняя

### ✅ Исправлено: .gitignore — отсутствовал dist-server
- **Файл:** `.gitignore`
- **Проблема:** Директория `dist-server` (временные файлы SSR-сборки) не была в `.gitignore`.
- **Исправление:** Добавлена `dist-server`.
- **Серьёзность:** 🟢 Низкая

## 3. КОД И АРХИТЕКТУРА

### ✅ Исправлено: Мёртвый код — комментарии EmailJS
- **Файл:** `components/Contact.tsx`
- **Проблема:** 12 строк комментариев с инструкцией по настройке EmailJS, хотя проект использует Resend.
- **Исправление:** Удалены устаревшие комментарии.
- **Серьёзность:** 🟢 Низкая

### ✅ Исправлено: console.log/console.error в продакшен-коде
- **Файлы:** `components/ErrorBoundary.tsx`, `components/PWAUpdateToast.tsx`, `lib/hooks/useContactForm.ts`
- **Проблема:** `console.error` и `console.log` в продакшен-коде засоряют консоль браузера и могут раскрывать внутреннюю информацию.
- **Исправление:**
  - `ErrorBoundary`: заменено на отправку события в Google Analytics через `gtag('event', 'exception')`.
  - `PWAUpdateToast`: удалены `console.log` из callback-ов регистрации Service Worker.
- **Серьёзность:** 🟡 Средняя

### ✅ Исправлено: Ошибка валидации Turnstile перезаписывала ошибку телефона
- **Файл:** `lib/hooks/useContactForm.ts`
- **Проблема:** Если пользователь не проходил Turnstile, ошибка записывалась в поле `errors.phone`, перезаписывая реальную ошибку валидации телефона.
- **Исправление:** Ошибка Turnstile теперь записывается в отдельное поле `errors.captcha` и отображается отдельно над виджетом.
- **Серьёзность:** 🟡 Средняя

### ✅ Исправлено: Аватары в отзывах без SkeletonImage
- **Файл:** `components/Testimonials.tsx`
- **Проблема:** Аватары клиентов загружались через обычный `<img>` без placeholder-эффекта, что создавало визуальный скачок при загрузке.
- **Исправление:** Заменено на `SkeletonImage` — с shimmer-эффектом и lazy loading.
- **Серьёзность:** 🟢 Низкая

### ✅ Исправлено: performance.css — агрессивный will-change
- **Файл:** `performance.css`
- **Проблема:** Глобальные правила `will-change: transform` и `will-change: opacity` на всех элементах с hover-эффектами создавали избыточное потребление памяти. Браузер выделяет отдельные слои GPU для каждого такого элемента, что на мобильных устройствах может привести к падению производительности.
- **Исправление:** Удалены глобальные `will-change` правила. Framer Motion сам управляет GPU-ускорением через `transform` и `opacity`.
- **Серьёзность:** 🟡 Средняя
