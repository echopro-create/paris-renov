# Обновленный план реализации SEO

Этот план основан на анализе текущего состояния проекта и задачах по улучшению SEO и доступности.

## 1. Техническое SEO и Метаданные

### index.html
- [ ] **Настройка GA4**: Заменить плейсхолдер `%VITE_GA_ID%` на реальный ID или настроить корректную подстановку через `.env`.
- [ ] **Удаление дублей**: Удалить дублирующую ссылку `apple-touch-icon` (конфликт с `vite-plugin-pwa`).
- [ ] **Канонический URL**: Убедиться, что `https://atelier-alexei.fr/` указан как канонический домен.
- [ ] **Preload**: Проверить и синхронизировать `fetchpriority="high"` для предзагрузки Hero-изображения.

### Sitemap и Robots
- [ ] **Sitemap**: Проверить актуальность `sitemap.xml`.
- [ ] **Robots**: Проверить корректность ссылки на sitemap в `robots.txt`.

## 2. Контент и Семантическая Структура

### Изображения и Alt-тексты
- [ ] **Services**: Уточнить alt-тексты для изображений услуг (сделать их более специфичными, например, "Ремонт ванной комнаты в Париже" вместо "Service de rénovation").
- [ ] **SkeletonImage**: Улучшить атрибут `sizes` в компоненте `SkeletonImage` или передавать его пропсом для загрузки оптимального размера изображения.

### Доступность (ARIA)
- [ ] **Contact**: Добавить атрибуты `aria-invalid` и `aria-describedby` для полей ввода формы для улучшения доступности валидации.
- [ ] **LocalBusiness Schema**: В `index.html` дополнить схему `GeneralContractor`:
    - Добавить массив `sameAs` со ссылками на социальные сети.
    - Проверить и актуализировать `priceRange` и часы работы.

## 3. Производительность (Core Web Vitals)

### Оптимизация
- [ ] **CSS**: Проверить `performance.css` на предмет `content-visibility: auto`, чтобы избежать сдвигов макета (CLS).

## 4. Локальное SEO
- [ ] **Контент**: Внедрить ключевые слова "Paris" и названия районов в описания услуг и проектов, где это уместно.
